思路-> 先学list在学set,最后看list的底层原理

收获->性质以及定义;还有熟悉额外的打印输出利用stream流;学会在类中重写equal和hashcode这样在存入数据的时候可以保留hashcode的性质

当谈及 Java 集合框架时，主要分为 List 和 Set 两大类：

List

特性：
有序（元素按照插入顺序或自定义顺序排列）
可重复（可以包含相同的元素）
有索引（可以通过索引访问元素）

ArrayList

原理：
基于动态数组实现
支持快速随机访问和修改元素

LinkedList

原理：
基于双向链表实现,根据进入的顺序排序
支持高效的插入和删除操作，但访问元素需要遍历链表

Set

特性：
无序（元素没有特定顺序）
不可重复（不允许包含重复的元素）
无索引（不能通过索引访问元素）

HashSet

原理：
基于哈希表实现,存储实例的时候注意在类中重写equal和hashcode,这样才可以保证Hashset的不可重复特性
插入、删除、查找操作的平均时间复杂度为 O(1)

TreeSet

原理：
基于红黑树（自平衡二叉查找树）实现
元素按照自然顺序或者定制顺序进行排序

LinkedHashSet

特性：
继承自 HashSet
内部使用链表维护插入顺序，因此可以保持元素的插入顺序
这些类在 Java 集合框架中各自有着不同的实现方式和特点，适用于不同的数据存储和访问需求。

困惑->1 什么是IO流？2 字符流和字节流的区别？3 基本数据类型是什么流?4 流的基本性质

思路->逐一敲代码

遇到的困难->混乱概念

收获->初次面熟io

1 io流就是Input流和Output流 说人话就是输入输出流,用于内存和存储设备(外部设备)的交换

2 字符流就是用来处理文本，就是txt的，经典的类有比如Write和Read,读和写更接近人类的习惯，字符也是。
字节流就是都可以处理的,比如FileInputStream和FileOutputStream

3 基本数据类型的输入输出流是字节流，可以读写字节和字符还可以读写基本数据类型

4 有队列的基本性质 先进先出，还要满足文件的RO(只读)和WO(只写)
