当我尝试向HashSet里面添加重复的元素的时候:

在 Java 中，HashSet 是基于哈希表实现的，它本质上不允许存储重复元素，这是通过哈希表的性质决定的。即使你尝试重写 Comparator 方法，这对 HashSet 并没有影响，因为 HashSet 不依赖于元素之间的顺序或比较器。

具体来说：
HashSet 的特性：
HashSet 通过哈希函数确定元素的存储位置，不依赖于元素的排序或者比较器。
当你尝试添加重复元素时，HashSet 会忽略重复元素，因为它使用哈希表来确保元素的唯一性。
重写 Comparator 对 HashSet 的影响：

Comparator 接口主要用于定义对象之间的顺序关系，但 HashSet 并不关心元素的顺序或比较器。
即使你在元素类中实现了 Comparator 接口并重写了比较方法，这对 HashSet 没有任何影响。HashSet 仍然会根据哈希值来确定元素的存储位置，并且不会存储重复的元素。
因此，无论你如何重写 Comparator 方法，HashSet 都不会允许添加重复元素。如果你需要允许重复元素，并且希望元素有序，应该选择 TreeSet，并适当实现 Comparator 或确保元素类实现了 Comparable 接口。
